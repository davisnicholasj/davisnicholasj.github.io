Allergen Detection App

Scan ingredients. Get a Buy, Review, Do Not Buy, or Retake verdict — tailored to your allergen profile.

Food allergies create genuine anxiety at the grocery shelf — a single missed ingredient can cause a serious reaction. Most apps require manual ingredient searches; this app flips the experience so the label does the work.

Users log their allergens once, then scan any ingredient list to get a clear verdict: Buy, Review, Do Not Buy, or Retake Photo. The Review state handles cross-contamination advisories separately from direct ingredient hits — surfacing the exact label wording so the user can make an informed call rather than getting a false all-clear.

Problem
Manual ingredient scanning is slow and error-prone
Key decision
4-state verdict handles real-world label ambiguity
User flow
Log allergens → Scan label → Get verdict

The image is sent to a Supabase Edge Function, which calls the Google Gemini API for multimodal image analysis. Gemini handles both OCR and allergen assessment in a single pass, using the user's stored allergen profile as context — no separate ingredient database or rules engine required.

The four-state classifier (Buy / Review / Do Not Buy / Retake Photo) distinguishes between direct ingredient hits, cross-contamination advisories, and unreadable images. The app also resolves allergen aliases automatically — casein to dairy, semolina to gluten — so users don't need to know every ingredient name variant.

SCP Change Assessor

An AI reviewer embedded in CI/CD that risk-tiers proposed AWS policy changes and routes approvals automatically.

AWS Service Control Policy changes carry real organizational risk — a misconfigured deny can lock hundreds of accounts out of critical services. Manual review processes are slow and inconsistent; approval chains don't scale as the platform grows.

This tool embeds an AI assessor directly into the CI/CD pipeline. It reads proposed changes, compares them against existing policies, assigns a risk tier (Low / Medium / High / Critical), and automatically routes to the appropriate approvers based on risk profile — before anything reaches production.

Risk tiers
Low · Medium · High · Critical
Integration point
Runs inside existing CI/CD workflow, no new tooling required
Outcome
Right approvers, every time, automatically

The assessor is a Python service invoked as a CI/CD pipeline step. It diffs the proposed SCP JSON against the current policy state, then passes a structured prompt to an LLM containing the change delta, existing policy context, and a risk-scoring rubric.

The model returns a structured risk tier and rationale. The pipeline then consults an approver-routing config (Terraform-managed) to gate or auto-approve the change. High and Critical tiers block merge until human sign-off is received.
